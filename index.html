Ubuntu 14.04 802.1X

最近公司的无线网络出奇的慢，而有线网络必须使用802.1认证才能用，限制诸多。
更无语的是，在Ubuntu环境下怎么都用不了有线网络了。同样的设置，之前是可以连接上的。
搜索了半天，说是升级了network-manager的问题。我也跟着降级到network-manager_0.9.4，出现了很多问题，也没成功。
晚上时查看software center发现在上个月确实升级了network-manager-gnome的包。想到之前问过运维同学（在6、7月的时候是可以用无线的）

哈哈，把它降到0.9.8.8-0ubuntu4， 不要后面带4.3
中间出现了一些下面类似的错误：

dpkg：警告：downgrading libnm-gtk0 from 0.9.8.8-0ubuntu4.3 to 0.9.8.8-0ubuntu4
(正在读取数据库 … 系统当前共安装有 459583 个文件和目录。)
Preparing to unpack libnm-gtk0_0.9.8.8-0ubuntu4_amd64.deb …
Unpacking libnm-gtk0 (0.9.8.8-0ubuntu4) over (0.9.8.8-0ubuntu4.3) …
dpkg: dependency problems prevent configuration of libnm-gtk0:
libnm-gtk0 依赖于 libnm-gtk-common (= 0.9.8.8-0ubuntu4)；然而：
系统中 libnm-gtk-common 的版本为 0.9.8.8-0ubuntu4.3。

dpkg: error processing package libnm-gtk0 (–install):
依赖关系问题 - 仍未被配置
在处理时有错误发生：
libnm-gtk0

看来是需要把依赖的包也降级了，还好只用到这几个包：

  • network-manager-gnome_0.9.8.8-0ubuntu4_amd64
  • libnm-gtk0
  • libnm-gtk-common

这些都在 http://packages.ubuntu.com/trusty/ 的“网络”里可以下载到。
下载后执行

sudo dpkg -i libnm-gtk0_0.9.8.8-0ubuntu4_amd64.deb libnm-gtk-common_0.9.8.8-0ubuntu4_all.deb
sudo dpkg -i network-manager-gnome_0.9.8.8-0ubuntu4_amd64.deb network-manager_0.9.8.8-0ubuntu7_amd64.deb

现在还只是在家试的，但愿明天到公司可用。

常用的shell整理

在微博上关注了@linux命令行精选网 ，每天都会发布commandlinefu.cn 比较有意思也很常用的shell命令。这里转发我常用到的，学习并整理。

文本编辑

  1. sort排序

    1. 读取文件并排序，删除重复的条目，把新的内容输出到新文件里。

      sort -u filename > newfilename

  2. awk

    1. awk ‘!x[$0]++’ 删除重复的行 (不需要排序)
    2. awk ‘{print NR,$0}’ 显示行数
      这个直接可以wc -l file嘛
    3. awk ‘sub(“$”, “\r”)’ unixfile.txt > winfile.txt 用 awk实现 unix2dos命令
  3. sed
    1. sed -n ‘1~5{N;N;p}’ file.txt 每5行打印其中3行
    2. sed -i ‘s/oldname@example.com/newname@example.com/g’ grep oldname@example.com -rl . 多个文件替换邮件地址
    3. sed -i ‘s/\r//‘ 转化为unix格式

  4. find

    1. find . -type f -size -80k -delete 查找并删除小于指定大小的文件
    2. find -name “.tgz” -or -name “.tar.gz” | while read file; do echo “$file: “; tar -tzf $file; done 列出所有tar 里边的文件
    3. find . -type f | sed -n ‘s/..*.//p’ | sort -f | uniq -ic 按扩展名清点文件
    4. find . -type f -print0 | xargs -0 perl -pi.save -e ‘tr/A-Z/a-z/‘ 递归的转小写
    5. find . -type f -exec ls -l –full-time {} + | sort -k 6,7 多目录中按日期排序
    6. find -type f | grep -v “^./.git” | xargs md5sum | md5sum 查找文件并计算md5

  5. file and folder

    1. test -d folder || mkdir folder 没有再建目录
    2. while true; do du -s ; sleep ; done 监控文件大小

系统

  1. history保存命令的执行时间， 最好是把它加到.bashrc里

    export HISTTIMEFORMAT="%F %T "

  2. :!bash 在vi会话中执行bash，可以执行多个命令，而使用!命令的方式来每次只能执行一条命令，输入exit和回车来返回vi

  3. apt-cache rdepends 查看包的依赖

Kafka学习笔记

Kafka is a distributed, partitioned, replicated commit log service. It provides the functionality of a messaging system, but with a unique design.

####概念和术语###

  • Message，消息，是指在生产者、存储端和消费者之间传输数据。
  • Broker， 负责存储消息的存储阵列。
  • Producer，负责产生消息并发送给Broker。
  • Consumer，负责消息的消费。
  • Topic，由用户定义并在Broker上配置。producer发送消息到某个topic下，consumer从某个topic下消费消息。
  • partition，可以把一个topic分为多个分区。每个分区是一个有序，不可变的，顺序递增的commit log
  • Consumer Group，由多个消费者组成，共同消费一个topic下的消息，每个消费者消费部分消息。这些消费者就组成一个分组，拥有同一个分组名称,+ 通常也称为消费者集群
  • Offset，分区中的消息都有一个递增的id，我们称之为Offset。它唯一标识了分区中的消息。

Kafka环境搭建

Kafka单机环境搭建比较简单，官方文档quickstart 说的也比较详细。下面我把我的安装步骤记录如下：

  • Step 0：安装JAVA环境。Kafka是用Scala编写的，可以直接运行在JVM上。我们不需要单独去安装Scala运行环境，只需部署Java环境就OK。（一般都会有了吧），我这里用的是JDK1.7。

    yangqinlong@BJ-yangqinlong:~$ java -version
    java version “1.7.0_55”
    Java(TM) SE Runtime Environment (build 1.7.0_55-b13)
    Java HotSpot(TM) 64-Bit Server VM (build 24.55-b03, mixed mode)

下面的步骤基本和quickstart基本一致了

  • Step 1: Download the code
    下载0.8.1.1版本 并解压。
    $ tar -xzf kafka_2.10-0.8.1.1.tgz
    $ ln -s kafka_2.10-0.8.1.1 kafka # 我放到了/opt目录下，起个别名
    $ cd kafka

  • Step 2: Start the server
    先启动zookeeper， （加上&放到后台去执行）
    $ bin/zookeeper-server-start.sh config/zookeeper.properties &
    然后启动kafka server， （加上&放到后台去执行）
    $ bin/kafka-server-start.sh config/server.properties &

  • Step 3: Create a topic
    $ bin/kafka-topics.sh –create –zookeeper localhost:2181 –replication-factor 1 –partitions 1 –topic test
    bin/kafka-topics.sh的参数很多， zookeeper是必须的，create时 replication-factor、partitions、topic是必须的。其它参数后面用到时再介绍。
    查看Topic：
    $ bin/kafka-topics.sh –list –zookeeper localhost:2181

  • Step 4: Send some messages
    $ bin/kafka-console-producer.sh –broker-list localhost:9092 –topic test
    输入完这条命令就可以输入要发送的消息了

    Hello world, Kafka
    Hello, Kafka

  • Step 5: Start a consumer
    $ bin/kafka-console-consumer.sh –zookeeper localhost:2181 –topic test –from-beginning
    新打开一个终端来执行上面一条命令，就可以看到之前发送的消息了
